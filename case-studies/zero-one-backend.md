# ZERO-ONE: 자동 매칭과 학습 흐름 백엔드

- 역할: 8인 팀의 Backend Contributor, 이후 5인 피봇 팀의 Backend Lead
- 공개 서비스: https://zeroone.it.kr
- 저장소: 비공개 팀 저장소. 이 문서는 공개 가능한 설계 수준의 증빙입니다.

## 담당 범위

개발자 스터디 매칭 서비스에서 선호 주제·가능 시간대·학습 성향을 벡터화하고, K-Means와 거리 기반 그룹화로 유사한 학습자를 묶는 흐름을 설계했습니다. 같은 조합이 계속 반복되지 않도록 그룹 내부에서는 누적확률분포 기반 가중 선택을 사용했습니다.

주기 실행은 Spring Batch와 Scheduler로 연결했습니다. 매칭 이후 스터디 공간 생성, 참여자 등록, 출석 초기화가 이어지는 도메인 흐름을 하나의 일관된 처리 단위로 다뤘습니다. 이후 서비스가 바이브코딩 학습 지원 플랫폼으로 피봇될 때는 학습 흐름·피드백·커뮤니티 중심의 백엔드 전환을 리드했습니다.

## 공개 범위와 한계

- 팀 저장소는 비공개이므로 직접 GitHub 링크를 제공하지 않습니다. 공개 서비스와 이 기술 문서로 역할·설계 선택을 검증할 수 있게 했습니다.
- 사용자 데이터, 운영 토폴로지, 내부 배치 설정, 구현 코드는 공개하지 않습니다.
- 이 문서는 성능 수치를 주장하지 않습니다. 면접에서는 보안 범위 안에서 도메인 경계, 배치 실행 순서, 정합성 판단을 설명할 수 있습니다.
